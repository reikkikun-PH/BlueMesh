# Customization Rules

## GitHub Release Changelogs
- Always format GitHub release changelogs using clear bullet points.
- Explain the improvements in a positive, user-centric way.
- Use plain, non-technical language that general users can easily understand (e.g., describe the real-world impact, such as "avoids messages getting stuck on send" instead of technical terms like "GATT/MTU attribute size capping").
